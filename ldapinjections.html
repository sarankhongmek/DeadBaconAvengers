LDAP injection

LDAP injection is use to compromise the authentications process used by websites applications. Websites that construct Lightweight Directory Access Protocol (LDAP) statements from data provided by users are vulnerable to this type of attack.

LDAP directories stores information like people personal information, user name and password, servers and devices that is connected to it. If the directory is used for website authentications, the attacker can execute malicious code into a user input field, gain accesses to user directory and edit their user profile details.

LDAP injections works the same as SQL injections, a type of exploit that uses SQL code to inject in web forms. This types attack occurs when weak validations is put in place.

Security experts say that the main reason for such attack works is that their security system is not put in place during development process. To protect the integrity of Web sites that use forms, experts recommend implementing a sanitization process to control the types and numbers of characters that can be accepted by input boxes as well as the use of multi-factor authentication (MFA) for public-facing web applications.
